# 311: I was a 10x engineer. Now I'm useless.

> **Creator**: Mo Bitar | **Platform**: YouTube | **Date**: 2026-03-07 | **Duration**: 12:29

## Summary

Mo Bitar, a self-described former "10x engineer," reflects on how heavy AI tool usage has degraded both his coding ability and his emotional relationship with software development. After successfully using Codex (GPT-4.5) to autonomously deploy his app Shape.work — giving it AWS CLI and GitHub API access with no code review — he finds himself unable to care about the result. The product works, but he feels no ownership, pride, or ability to sell it authentically.

The deeper argument is that the craft of software development was never purely about the output — it was about the struggle, growth, and identity that came from building things by hand. AI has short-circuited that feedback loop entirely. Bitar compares vibe coding to evolution's blind trial-and-error process: statistically effective but incomprehensible. The problem isn't that AI produces bad code — it's that it produces code the developer has no relationship to.

He ends in a kind of paralysis: he can't review AI code fast enough to maintain quality, he can't regain his hand-coding skills without going cold turkey on AI tools, and he can't sell something he has no emotional investment in. He frames AI coding tools as a drug with no viable middle ground — the only recovery is total abstinence, which he's unwilling to commit to given competitive and livelihood pressures.

---

## Key Concepts

### Skill Atrophy Through Tool Dependency
Bitar describes a concrete cognitive regression: after sustained AI tool use, his brain actively resists manual coding by asking "why would you do this by hand?" The ease of the AI shortcut has eroded not just skill but motivation to apply skill. This parallels GPS dependency reducing spatial navigation ability — the tool replaces not just effort but the neural pathways that made effort possible.

### The Emotional-Craft Connection in Software
Bitar argues that developers build emotional investment in software through struggle and suffering — the overnight uncertainty, the morning breakthrough. This process also develops professional identity and selling conviction. AI-generated code bypasses this entirely, producing what he calls a "hot dog": functional, calorie-equivalent, but soulless. Without that bond, he cannot authentically market or support the product.

### Vibe Coding as Evolution Without Intent
He extends his earlier analogy: natural selection produces staggeringly complex organisms through random variation + survival filtering, with no top-down design. Vibe coding works similarly — LLM generates variations, tests act as survival criteria, passing code accumulates. The output can be impressive, but the codebase is "incomprehensible and inscrutable," exactly like DNA. This isn't a bug, it's the mechanism.

### The Drug Dependency Model for AI Tools
Bitar rejects the "use AI for boring tasks, code hard parts yourself" compromise. In practice, after doing hard work manually, the 10-minute AI shortcut for the next step feels so rewarding that it pulls you back in. He argues there is no sustainable partial use — the reinforcement gradient always favors more AI, not less. The only viable alternative is complete removal, analogous to addiction recovery.

### Developer Identity Crisis in the AI Transition
He raises the question of what engineering credentials and career resilience mean now. Developers previously had a security floor: skills were durable, sabbaticals were safe, re-entry was guaranteed. That premise may no longer hold. Combined with uncertainty about whether LLMs will plateau or keep improving, he characterizes the whole developer community as being in a "liminal waiting space" — neither adapted nor abandoned, just holding their breath.

## Practical Takeaways

- **No-review autonomous deployment is real and works**: Bitar confirmed that giving an agent (Codex/GPT-4.5) access to AWS CLI + GitHub API with a pure outcome prompt ("get users to a working download page") can successfully deploy a production app end-to-end — useful data point for evaluating agentic pipelines.
- **Survival criteria / outcome-framing is an effective agent directive**: Defining what "done" looks like in user-observable terms (not technical specs) appears to give agents sufficient direction for complex multi-step deployment tasks.
- **Consider the emotional sustainability of your workflow**: If you cannot authentically represent and sell what you build, the workflow has a hidden cost that compounds over time — especially for founders and indie developers who must embody their product.
- **Partial AI use may be cognitively unstable**: The incremental reward structure of AI tools creates pressure toward full dependency; teams should design deliberate friction or AI-free zones if skill maintenance matters.
- **Code review velocity is a real constraint**: Bitar's observation that developers cannot review AI-generated code fast enough to keep pace is a genuine bottleneck — teams relying on human review as a quality gate may need to rethink validation strategies entirely.
