## Details

### The $690 Billion Infrastructure Sprint vs. the $35 Billion Revenue Reality

The five largest US hyperscalers — Amazon ($200B), Alphabet ($175-185B), Microsoft ($120B+), Meta ($115-135B), and Oracle ($50B) — have collectively committed $660-690 billion in 2026 capital expenditure, nearly doubling 2025 levels. This is driven by what Futurum calls "a shared conviction that AI workloads will consume every available unit of compute capacity." Meanwhile, the entire cohort of pure-play AI vendors — OpenAI (~$20B ARR), Anthropic (~$9B run rate), and everyone else — likely accounts for under $35B in projected 2026 revenue. The timing gap is 18-36 months before infrastructure generates proportional returns, power constraints are becoming a binding limitation, and the efficiency paradox (cost reductions increasing total consumption) may require even more infrastructure. This is the macro-level version of the spending-without-returns pattern that Griffonomics ([065](../sources/065-griffonomics-saaspocalypse.md)) and Nate B Jones ([059](../sources/059-nate-b-jones-ai-spending-skills.md)) have been tracking at the industry level.

**Sources**: [Futurum: AI Capex 2026](https://futurumgroup.com/insights/ai-capex-2026-the-690b-infrastructure-sprint/), [Gartner: $2.5T AI Spending](https://www.gartner.com/en/newsroom/press-releases/2026-1-15-gartner-says-worldwide-ai-spending-will-total-2-point-5-trillion-dollars-in-2026), [Goldman Sachs: $500B+](https://www.goldmansachs.com/insights/articles/why-ai-companies-may-invest-more-than-500-billion-in-2026)

### "Developers Have Stopped Writing Code" — Claims vs. Context

Fortune's February 13 piece aggregates the most aggressive "developers abandoned programming" claims in one article. Spotify co-CEO Gustav Söderström says the company's best developers "have not written a single line of code since December." Boris Cherny, head of Claude Code at Anthropic, says he hasn't written code in over two months. OpenAI claims GPT-5.3-Codex "was instrumental in creating itself." The framing is triumphant, but Gary Marcus calls it "weaponized hype" lacking supporting data for complex application development. The reality heuristic from our collection applies: bounded, verifiable tasks (bug fixes, feature additions via Slack) succeed, while open-ended architectural work still requires human judgment. Spotify shipping 50+ features via AI workflows is real. "Developers don't write code anymore" is marketing. The gap between these statements is where the actual story lives.

**Source**: [Fortune: OpenAI and Anthropic spark coding revolution](https://fortune.com/2026/02/13/openais-codex-and-anthropics-claude-spark-coding-revolution-as-developers-say-theyve-abandoned-traditional-programming/)

### Agent Security: 88% Incident Rate, 14% Pre-Deployment Approval

Gravitee's State of AI Agent Security 2026 report quantifies what IBM Technology ([057](../sources/057-ibm-zero-trust-ai-agents.md)) described theoretically: the gap between agent deployment velocity and security controls. 88% of organizations report confirmed or suspected AI agent security incidents in the past year (92.7% in healthcare). Only 14.4% have complete security approval for their full agent fleet before deployment. Organizations actively monitor just 47.1% of their AI agents. Only 21.9% treat AI agents as independent identity-bearing entities — the exact problem IBM's zero trust framework addresses. The shared API key problem (45.6% of organizations) is the credential management anti-pattern that IBM identified as an "absolute no-no." Three CVEs in Anthropic's own Git MCP server (CVE-2025-68145, CVE-2025-68143, CVE-2025-68144) enable remote code execution via prompt injection — directly validating the MCP supply chain concerns raised by ThePrimeagen ([017](../sources/017-primeagen-skills-security.md)).

**Sources**: [Gravitee: State of AI Agent Security 2026](https://www.gravitee.io/blog/state-of-ai-agent-security-2026-report-when-adoption-outpaces-control), [Adversa AI: MCP Security Resources](https://adversa.ai/blog/top-mcp-security-resources-february-2026/), [Dark Reading: Agentic AI Attack Surface](https://www.darkreading.com/threat-intelligence/2026-agentic-ai-attack-surface-poster-child)

### Claude Code February Updates

Claude Code shipped several significant features in February 2026. Agent teams (research preview) enables multi-agent collaboration within Claude Code sessions — requiring CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1 to enable. Automatic memory recall allows Claude to record and recall memories across sessions without explicit user configuration. The skill character budget now scales with context window (2% of context), addressing the skill truncation issue that Simon Scrapes ([062](../sources/062-simon-scrapes-claude-code-levels.md)) noted. A critical Bash tool fix resolves "Bad substitution" errors when heredocs contain JavaScript template literals like `${index + 1}` — a bug that affected agent workflows processing generated code. Skills from `--add-dir` directories are now loaded automatically.

**Source**: [Claude Code Changelog](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md)

## Also Ingested

- **[071: Future of Software Development — Deer Valley Retreat Fragments](../sources/071-martin-fowler-future-software-dev.md)** — Martin Fowler publishes fragments from a ~50-person retreat (Thoughtworks employees, industry experts, clients) on AI's impact on software development. The most important contribution is **Cognitive Debt** — a structured framework for the knowledge gap that accumulates when teams outsource understanding to LLMs: "Either we pay interest through harder changes, or pay principal through explicit knowledge investment." Drew Breunig's "spec-as-library" experiment (500-line markdown spec + 500-line YAML test suite, no code, implemented across 7 languages by LLMs) pushes the specification-first thesis to its logical extreme. Laura Tacho's observation that "Developer Experience and Agent Experience intersect completely" generalizes the AI-friendly codebase prerequisite. Career vulnerability distribution is counterintuitive: mid-level developers face the greatest disruption, not juniors. HBR research shows AI-assisted workers expanding scope and hours without being asked, leading to burnout — the first concrete data suggesting productivity gains may be partially offset by unsustainable human work patterns.
