# /state-of-field

Return the current validated state of the AI engineering field, drawn from research-backed thesis validation.

## Workflow

1. Read `STATE.md` at the repository root.
2. Parse the YAML frontmatter to confirm the last research run date and coverage statistics.
3. If a **topic filter** is provided as an argument:
   - Search the Key Theses section for theses matching the filter keyword(s) (case-insensitive match against title, summary, and module references).
   - Return only the matching theses with their full metadata (status, confidence, trend, evidence, modules, summary).
   - If no theses match directly, search the Urgent Updates section for relevant items.
   - If still no matches, report that no theses match the filter and suggest related topics from the thesis list.
4. If **no filter** is provided:
   - Return the Executive Summary.
   - Return all Key Theses in their structured format.
   - Return the Urgent Updates section.
   - Omit the Methodology Notes section for brevity (mention it is available if needed).
5. Format output for consumption by other skills:
   - Use the exact structured thesis format (Status/Confidence/Trend/Evidence/Modules/Summary) so downstream skills can parse fields programmatically.
   - When returning filtered results, include the total thesis count and research date for context.

## Output Format

```
## State of AI Engineering (as of {generated date})
Modules validated: {N} | Concepts reviewed: {N}

### Executive Summary
{summary text}

### Matching Theses ({N} of {total})
{thesis entries in structured format}

### Urgent Updates
{relevant updates}
```

## Examples

- `/state-of-field` -- Full summary with all theses
- `/state-of-field bubble` -- Theses related to the AI bubble, financial distress, or market dynamics
- `/state-of-field enterprise` -- Theses about enterprise adoption, productivity, and deployment reality
- `/state-of-field security` -- Theses about prompt injection, agent security, and attack surfaces
- `/state-of-field infrastructure` -- Theses about power constraints, compute scaling, and capex

## Dependencies

- Requires `STATE.md` at repository root (generated by `/research-curriculum` pipeline)
- STATE.md is regenerated when research validation runs are completed across all 6 curriculum modules
